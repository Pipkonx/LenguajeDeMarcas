Aplicaciones

La barra de herramientas de Visual Studio Community es una interfaz que contiene varios botones y opciones rápidas para realizar tareas comunes durante el desarrollo de software. Por lo general, incluye:

  1. Archivo: Para crear, abrir, guardar o cerrar proyectos y soluciones.
  2. Edición: Herramientas para deshacer, rehacer, copiar, pegar, buscar y reemplazar.
  3. Ver: Controla las ventanas y paneles del IDE, como el Explorador de soluciones y la Terminal.
  4. Git: Opciones para gestionar el control de versiones con Git, como hacer commit, push, pull y ver el historial de cambios.
  5. Proyecto: Opciones para gestionar la configuración y las propiedades del proyecto.
  6. Compilar: Opciones para compilar y construir tu proyecto, asegurando que el código sea transformado en un ejecutable o librería.
  7. Depuración: Herramientas para iniciar, detener y depurar el código..
  8. Prueba: Herramientas para ejecutar pruebas unitarias, gestionar los resultados de las pruebas y asegurar que tu código funciona correctamente.
  9. Analizar: Opciones para analizar el rendimiento del código, el uso de memoria y otras métricas que pueden ayudar a mejorar la calidad y eficiencia.
  10. Herramientas: Configuraciones y extensiones para personalizar el entorno de desarrollo.
  11. Extensiones: Para gestionar y agregar extensiones y complementos que amplían las funcionalidades de Visual Studio.
  12. Ventana: Permite gestionar las ventanas abiertas dentro del IDE, como mover, organizar o cambiar entre ellas.
  13. Ayuda: Acceso a la documentación, recursos y soporte

Archivo

  1. Nuevo: Crear proyectos, soluciones y archivos nuevos.
  2. Abrir: Abrir proyectos, soluciones o archivos existentes
  3. Guardar: Guardar cambios en proyectos y archivos
  4. Cerrar: Cerrar archivoso proyectos abiertos
  5. Salir: Cerrar visual Studio

Editar

  1. Deshacer/Rehacer: Deshacer o rehacer cambios en el código.
  2. Buscar y reemplazar: Buscar palabras o fragmentos de código y reemplazarlos.
  3. Copiar/pegar: Copiar y pegar código entre archvios o secciones.
  4. Ir a línea/símbolo: Navegar rápidamente a una línea específica o símbolo en el código

Ver

  1. Explorador de Soluciones: Ver y gestionar archivos y proyectos dentro de la solución
  2. Ventanas: Adminsitrar ventanas como el Terminal, el Explorador de Archivo, y la ventana de Errores.
  3. Diseñadores: Acceder a herramientas visuales para diseñar interfaces de usuario, si aplicable.

Git

  1. Commit: Hacer un commit de los cambios realizados al repositorio.
  2. Push/Pull: Subir o bajar cambios del repositorio remoto.
  3. Ver historial: Consultar el historial de commit de tu repositorio
  4. Branching: Crear y gestionar ramas para manejar diferentes versiones de tu proyecto.

Proyectos

  1. Propiedades: Configurar propiedades del proyecto, como compilación y dependencias.
  2. Configuraciones: Establecer configuraciones de compilación para diferentes entornos
  3. Agregar Elemento: Añadir nuevos archivos o recursos al proyecto.

Compilar

  1. Compilar: Compilar el código fuente para generar los archivos ejecutables.
  2. Compilación en segundo plano: Permite compilar sin bloquear el flujo de trabajo principal
  3. Compilación limpia: Elimina archivos generados previamente ante de hacer una nueva compilación

Depurar

  1. Iniciar Depuración: Ejecutar el códgio en modo depuración para detectar errores.
  2. Puntos de interrupción: Pausar la ejecución del código en lugares específicos para inspeccionar el estado
  3. Inspeccionar Variables: Ver los valores de variable y objetos en el código en tiempo de ejecución

Prueba

  1. Ejecutar Pruebas: Ejecutar pruebas unitarias para verificar que el código funciona correctamente.
  2. Ver Resultados: Consultar los resultados de las pruebas realizadas.
  3. Cobertura de Pruebas: Medir la cobertura de código durante las pruebas para asegurar que el código está adecuadamente probado.

Analizar

  1. Análisis de código: Inspeccionar el código en busca de problemas de rendimiento, errores y mejores prácticas.
  2. Análisis de rendimiento: Identificar cuellos de botella o áreas donde se puede mejorar la eficiencia del código.

Herramientas

  1. Administrador de Paquetes NuGet: Gestionar dependencias y paquetes de código reutilizables.
  2. Opciones: Configurar el entorno de desarrollo según tus preferencias personales.
  3. Perfil de rendimiento: Medir el uso de recursos como CPU y memoria durante la ejecución.

Extensiones

  1. Instalar/Desinstalar: Gestionar extensiones y plugins para agregar nuevas funcionalidades.
  2. Marketplace: Buscar y acceder a nuevas extensiones disponibles.

Ventana

  1. Organizar Ventanas: Cambiar el diseño de las ventanas, moverlas o anclarlas en diferentes posiciones.
  2. Cambiar entre ventanas: Alternar rápidamente entre las diferentes ventanas del IDE.

Ayuda

  1. Documentación: Acceder a la ayuda en línea y guías de Visual Studio.
  2. Foros y Comunidad: Buscar ayuda en la comunidad de desarrolladores.
  3. Acerca de: Información sobre la versión de Visual Studio y su licencia.
